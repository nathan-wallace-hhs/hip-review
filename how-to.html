How to Use the HIP Review Tool

Understand the prompt, required inputs, expected outputs, and copy the reusable LLM prompt.

What this tool does

The HIP Review tool helps teams review UX content audit reports in reports/ and compare insights across pages. Use the dashboard for analysis and use this guide when preparing consistent LLM-based audits.

Prompt

Use the standard "UX Content Audit Prompt — High Impact Page Review" template when generating audits so findings are structured consistently across the 50-page program.

Inputs you need before running the prompt

  • Page details: URL, title/purpose, user tasks, agency goal, audit date, reviewer, and audit ID.
  • Google Analytics (last 90 days): traffic, engagement, exits, scroll depth, referrers, devices, events, and search terms.
  • Siteimprove: accessibility issues by severity, readability score, link/spelling/SEO/policy checks, and DCI sub-scores.
  • Heatmaps: click behavior, scroll reach, rage/dead clicks, hotspots, and mobile/desktop differences.
  • Optional: USWDS/M-24-08 notes, prior feedback, and Search Console queries.

Outputs you should expect

  • A complete page audit with executive summary, quantified snapshot table, and six-dimensional findings.
  • Prioritized recommendations ranked by impact vs. effort with named owners.
  • Suggested before/after content revisions for the top changes.
  • Open questions and data gaps for anything that cannot be validated.
  • Appendix-ready references so each claim can be traced to data.

How to use this with the dashboard

  1. Run an audit with the copied prompt and your page-specific inputs.
  2. Save the result in reports/ as JSON or Markdown.
  3. Add the report metadata to reports/index.json.
  4. Open the dashboard to search, filter, and compare report trends.
